Quarto e RStudio: primeiro documento

Crie um arquivo .qmd e renderize-o em HTML

Neste tutorial, você criará seu primeiro documento Quarto dentro do seu Projeto RStudio, configurará seu cabeçalho YAML, escreverá um texto curto e produzirá uma página HTML:

arquivo .qmd → Render → documento HTML
ImportanteResultado esperado

Ao final, o arquivo primeiro-documento.qmd estará salvo no seu Projeto RStudio e o RStudio terá gerado e aberto primeiro-documento.html.

Antes de começar

Conclua o tutorial Git e GitHub e abra no RStudio o Projeto RStudio criado a partir do repositório individual. O nome desse projeto deve aparecer no canto superior direito da janela.

Não é necessário instalar nada novo: a instalação e a verificação do Quarto já fazem parte da Preparação do ambiente computacional.

1. Criar um documento Quarto

  1. No RStudio, escolha File → New File → Quarto Document.
  2. Na janela de criação, preencha:
    • Title: Primeiro documento;
    • Author: seu nome completo;
    • Format: HTML.
  3. Clique em Create.

O RStudio abrirá um novo arquivo com extensão .qmd no painel Source. Esse arquivo reúne o texto do documento e as opções de geração do resultado.

2. Configurar o cabeçalho YAML

O cabeçalho YAML fica no início do arquivo, entre duas linhas de três hífens (---). Ele informa ao Quarto o que é o documento e como ele deve ser produzido. Substitua o cabeçalho criado pelo RStudio pelo bloco abaixo:

---
title: "Primeiro documento"
author: "Seu nome completo"
lang: pt-BR
format:
  html:
    toc: true
---

Faça três verificações antes de continuar:

  • substitua Seu nome completo pelo seu nome;
  • mantenha as duas linhas ---, uma no começo e outra no fim do cabeçalho;
  • use espaços, e nunca a tecla Tab, antes de html: e toc:.

As linhas title, author e lang definem propriedades diretas do documento. Já html está dentro de format, e toc: true está dentro de html; por isso essas duas linhas são recuadas. Essa estrutura gera um documento HTML com sumário automático.

DicaYAML exige atenção à estrutura

Um cabeçalho sem a linha final ---, com uma indentação incorreta ou com Tab no lugar de espaços pode impedir a renderização. Quando isso acontecer, verifique primeiro o YAML antes de alterar o restante do texto.

3. Escrever o primeiro conteúdo

Deixe uma linha em branco após a linha final --- do YAML e substitua todo o conteúdo abaixo dela pelo trecho a seguir.

## Desempenho trimestral

Este documento apresenta uma síntese dos **resultados financeiros** do
segundo trimestre.

### Principais achados

- A receita cresceu 12%.
- Os custos permaneceram sob controle.
- A margem operacional aumentou.

| Indicador | Valor |
|:----------|------:|
| Receita   | 145   |
| Custos    | 92    |

4. Salvar dentro do Projeto RStudio

  1. Escolha File → Save.
  2. Salve o arquivo com o nome primeiro-documento.qmd na pasta principal do seu Projeto RStudio.
  3. Confirme, na aba Files, que o arquivo aparece nessa pasta.

Salvar dentro do projeto é importante: os arquivos da disciplina, o documento Quarto e os resultados gerados ficarão reunidos em uma única localização.

5. Renderizar em HTML

Com primeiro-documento.qmd aberto no painel Source, clique em Render, na barra superior do editor. Aguarde a conclusão do processo.

Renderizar transforma o arquivo-fonte .qmd em um documento final. Neste caso, o resultado é um arquivo HTML. O RStudio normalmente o abrirá no painel Viewer ou no navegador padrão.

6. Verificar o resultado

Confirme os três pontos abaixo antes de encerrar:

  1. o documento HTML mostra o título, seu nome, um sumário, os títulos de seção, a lista e a tabela;
  2. a aba Files contém primeiro-documento.qmd e primeiro-documento.html;
  3. não há mensagem de erro impedindo a renderização.

Se o resultado não estiver correto, volte ao arquivo .qmd, corrija-o, salve e clique em Render novamente. Você repetirá esse ciclo ao longo da disciplina.

DicaSe o botão Render não aparecer

Verifique se o arquivo aberto termina em .qmd e se você está no Projeto RStudio correto. Se o problema persistir, retorne à seção de verificação do Quarto na Preparação do ambiente computacional.

Próximo passo

Você concluiu o primeiro ciclo com Quarto: criou um .qmd, configurou seu YAML e gerou HTML. Continue com Markdown para documentos Quarto para aprender a estruturar melhor o corpo do documento. Nas atividades da disciplina, esse fluxo receberá gradualmente código R, citações, referências cruzadas, figuras e outros formatos de saída.
