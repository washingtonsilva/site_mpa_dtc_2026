Preparação do ambiente computacional

Windows: contas, instalação, configuração e validação antes da primeira aula

Roteiro de preparação

Conclua este roteiro antes da primeira aula. Ele prepara o computador para as atividades da disciplina.

NotaPlataforma adotada

As etapas detalhadas desta página foram preparadas para Windows. Usuários de macOS ou Linux devem utilizar os instaladores disponíveis nas páginas oficiais indicadas nas seções de R, RStudio, Quarto e Git.

  1. Crie uma conta gratuita no Overleaf e uma conta no GitHub.
  2. Instale R, RStudio, Quarto, TinyTeX e Git.
  3. Configure sua identidade no Git e verifique o Git Bash como terminal do RStudio.
  4. Faça a verificação do ambiente.
  5. Leia o tutorial Overleaf: primeiros passos e o tutorial prático de Git e GitHub para conhecer os fluxos de trabalho da disciplina.
  6. Quando forem publicados, crie seu repositório individual a partir do repositório-template e registre sua URL no cadastro de repositórios. O artigo de referência e o enunciado do miniartigo no Overleaf serão disponibilizados apenas na atividade correspondente.

Não há pré-requisito formal, mas as atividades serão registradas no GitHub desde a primeira aula. Quem ainda não conhece Git e GitHub deve estudar por conta própria com Happy Git and GitHub for the useR e com este roteiro. Não haverá um bloco de nivelamento durante as aulas.

Instalação dos programas

Linguagem R

DicaInstalar R 4.6.1
  1. Acesse CRAN.
  2. Clique em Download R for Windows e, na página seguinte, em base.
  3. Clique em Download R-4.6.1 for Windows.
  4. Após o download, execute o instalador e aceite as opções padrão até a instalação ser concluída.

RStudio

DicaInstalar o RStudio Desktop
  1. Acesse a página de download do RStudio.
  2. Na seção 2: Install RStudio, clique em Download RStudio Desktop for Windows.
  3. Após o download, execute o instalador e avance até a instalação ser concluída.

Sistema Quarto

DicaInstalar o Quarto CLI
  1. Acesse a página de download do Quarto.
  2. Baixe o instalador de Quarto CLI para Windows.
  3. Execute o instalador e avance até a instalação ser concluída.

TinyTeX

ImportanteInstalar o TinyTeX com antecedência

TinyTeX é a distribuição LaTeX recomendada para gerar PDF a partir de documentos Quarto. Faça a instalação depois de instalar o Quarto.

A instalação pode demorar, dependendo da conexão de internet e do computador. Execute-a com antecedência e aguarde sua conclusão antes da primeira aula.

No RStudio, abra a aba Terminal e execute:

quarto install tinytex

Se for necessário atualizar a distribuição posteriormente, execute:

quarto update tinytex

Git

DicaInstalar o Git for Windows
  1. Acesse git-scm.com e clique em Download for Windows.
  2. Após o download, execute o instalador.
  3. Mantenha as configurações padrão sugeridas. Na janela Adjusting the name of the initial branch, marque Override the default branch name for new repositories e informe main.
  4. Clique em Next nas janelas restantes e conclua a instalação.

Conta GitHub

Caso ainda não tenha uma conta, acesse github.com, clique em Sign up, conclua o cadastro e confirme seu e-mail. Use uma conta individual que você possa manter durante toda a disciplina.

Configurar o Git

Abra o Git Bash, disponível no menu Iniciar após a instalação do Git, e configure o nome e o e-mail que devem aparecer no histórico de seus commits:

git config --global user.name "Seu Nome"
git config --global user.email "seuemail@exemplo.com"

Use um e-mail associado à sua conta do GitHub. Para conferir as duas configurações, execute:

git config --global --get user.name
git config --global --get user.email

Git Bash como terminal do RStudio

Os comandos Git desta disciplina serão executados no Terminal do RStudio usando Git Bash. Depois de instalar o Git for Windows, o Git Bash pode já estar selecionado no RStudio, mas isso depende das configurações do seu computador. Verifique antes de alterar qualquer opção:

  1. Abra RStudio → Tools → Global Options → Terminal.
  2. Em New terminals open with, verifique a opção exibida.
  3. Se já estiver selecionado Git Bash, não altere nada. Caso esteja selecionado PowerShell ou outra opção, escolha Git Bash.
  4. Confirme a configuração e abra um novo terminal no RStudio.

Se Git Bash não aparecer na lista, reinstale o Git for Windows mantendo as opções padrão e tente novamente. O Git Bash padroniza os comandos com macOS e Linux. Ele oferece comandos no estilo Unix, mas não transforma o Windows em outro sistema operacional.

Verificação do ambiente

Antes da primeira aula, confirme que você consegue:

  1. Abrir R e RStudio.

  2. Abrir o terminal do RStudio e executar:

    quarto --version
git --version

  3. Confirmar que seu nome e e-mail do Git estão configurados conforme a seção anterior.

  4. Quando o repositório-template estiver disponível, criar seu repositório individual a partir dele, abri-lo como Projeto RStudio, confirmar que o Terminal usa Git Bash, editar um arquivo e usar git status, git add, git commit e git push para enviá-lo ao GitHub.

  5. Registrar a URL do seu repositório no cadastro da disciplina quando o formulário for publicado.
